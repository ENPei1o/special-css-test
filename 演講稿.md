## Slide 1: 標題頁
大家好！今天我們要探討的主題是「CSS 新紀元：流暢過渡與精準定位」 。隨著 Web 技術的演進，我們現在能透過 View Transitions 與 Anchor Positioning 這兩項 API，在網頁上打造出類原生 App 的流暢體驗 。


## Slide 2: 當前的開發痛點
在進入新技術之前，我們必須先面對目前的開發痛點：

動畫難度高：過去要實現跨頁面的流暢轉場，極度依賴複雜的 JavaScript 框架（如 Framer Motion），這不僅增加了學習曲線，也造成效能負擔 。


定位地獄 (Positioning Hell)：傳統的彈出式視窗常受到父層 overflow: hidden 的裁切或是 z-index 堆疊順序的干擾，導致難以實現可靠的定位 。

## Slide 3: 為何我們需要這兩項技術？
CSS 的原生解決方案正是為了克服上述困難而生：


View Transitions API：它讓 DOM 狀態的變化能像影片剪輯般無縫銜接，並由瀏覽器原生處理複雜的轉場邏輯 。


Anchor Positioning API：它讓元素能像磁鐵般互相吸附，擺脫 DOM 結構的限制，徹底解決層級與裁切問題 。

## Slide 4: View Transitions API - 基礎概念
這是一種瀏覽器原生的流暢轉場機制，其運作流程分為四個階段 ：


舊畫面快照：瀏覽器先擷取當前頁面的視覺狀態 。


DOM 更新：執行開發者提供的更新邏輯，例如內容切換 。


新畫面快照：更新後，立即擷取新頁面的視覺狀態 。


自動動畫：瀏覽器自動在兩個快照間生成平滑過渡，預設為淡入淡出 。

最核心的語法非常簡單：document.startViewTransition(() => { updateTheDOM(); }); 。

## Slide 5: View Transitions - 精準控制
想要創造如原生 App 般的「神奇移動」效果嗎？關鍵在於識別元素 ：


定義身份 (Identity)：透過 view-transition-name 為轉場前後的元素賦予跨越 DOM 的穩定「身份證」 。


自動計算：瀏覽器會根據身份自動計算位置與尺寸變化，產生補間動畫 。


自訂動畫：您還能透過偽元素如 ::view-transition-old 與 ::view-transition-new 來完全掌控舊狀態與新狀態的動畫細節 。

## Slide 6: 跨頁面過渡 (Multi-page Apps)
令人振奮的是，現在僅需一行 CSS 就能讓傳統多頁面網站（MPA）擁有流暢轉場 ：

@view-transition { navigation: auto; } 

這意味著完全無需 JavaScript 介入，瀏覽器就會自動處理跨頁面的視覺銜接 。這對於部落格文章切換、電商商品跳轉或是文件導航系統來說，能顯著提升連貫性與專業感 。





## Slide 7: CSS Anchor Positioning 解解定位難題
接著我們來看「錨點定位」。這是擺脫 DOM 結構束縛的革命性方案 ：


核心機制：它允許一個絕對定位元素與頁面上任何另一個元素（錨點）建立邏輯關聯，無視它們在 DOM 樹中的級別 。


解決裁切問題：過去為了避免 overflow: hidden，我們必須將元素移至 body 層級 。現在，即使元素在最外層，也能精準對齊深埋在結構中的錨點 。


原生優勢：純 CSS 解決、自動跟隨捲動、且由瀏覽器原生優化渲染效能 。



## Slide 8: Anchor Positioning - 語法實戰
實現定位僅需兩個標準步驟 ：



定義錨點：在觸發元素上使用 anchor-name: --my-btn; 。


關聯與定位：在目標元素上使用 position-anchor: --my-btn;，並透過 anchor() 函數（如 bottom: anchor(top);）來精確對齊 。

此外，還有 Position Area 語法，讓您能利用 9 格網格系統進行快速定位（如 position-area: top;） 。

## Slide 9: 動態自適應：Position Try
當空間不足時，瀏覽器能智慧翻轉定位元素 ：


智慧翻轉 (Intelligent Flipping)：透過 position-try-options: flip-block;，當 Tooltip 會被遮擋時，它會自動嘗試備用位置 。




告別 Popper.js：過去需要數十 KB 的 JS 套件才能實現的邊界檢測，現在僅需一行 CSS 。這對效能與維護性是巨大的提升 。


## Slide 10: 與 Top Layer 元素的完美整合
錨點定位是 Popover 與 Dialog 的最佳拍檔 ：


定位困境：頂層元素（Top Layer）雖然能無視 z-index，但傳統上難以與頁面中的觸發按鈕保持視覺關聯 。


解決方案：透過 position-anchor，這些頂層元素可以輕鬆「錨定」回觸發者，同時保留其置頂特性 。這非常適合用於下拉選單、教學引導或錯誤訊息提示 。



## Slide 11: 瀏覽器支援度與策略
雖然這兩項 API 非常強大，但我們仍需關注支援度 ：


支援狀況：Chrome 與 Edge 目前已全面支援，而 Safari 與 Firefox 則在開發階段 。




漸進增強 (Progressive Enhancement)：我們的核心哲學是首先為所有使用者提供基礎穩定的功能（如標準跳轉），再為現代瀏覽器使用者提供更流暢的進階體驗 。



功能檢測：可以使用 @supports 來確保僅在支援的瀏覽器中應用這些效果 。

## Slide 12: 實際應用場景與最佳實踐
在實踐中，請參考以下指南 ：


View Transitions：適用於電商圖片連續動畫、內容網站的文章切換、或儀表板的佈局重新排列 。




Anchor Positioning：適用於互動元件（Tooltip、下拉選單）、表單錯誤提示、以及資料視覺化的標註 。




專業建議：避免對大量元素同時使用轉場以優化效能，並務必檢測 prefers-reduced-motion 以尊重使用者的無障礙偏好 。



## Slide 13: 結論 - CSS 驅動的網頁未來
CSS 正在接管 UI 邏輯 。透過聲明式語法，我們能減少對重度 JS 框架的依賴，將密集計算交給瀏覽器原生引擎，提升效能與回應速度 。
